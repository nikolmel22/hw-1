Команди Git

  • git config --global user.name blabla - встановлює ім'я, яке буде відображатися
  • git config --global user.email blabal@bla - встановлює пошту, яка буде відображатися
  • git init- створює новий локальний репозиторій в поточній директорії
  • git status - перераховує всі нові або змінені файли, які потребують в фіксації
  • git add - додає файли в коміт. Зазвичай використовується "." або "*" для додавання всіх файлів. Щоб додати один файл потрібно вказати його ім'я та розширення
  • git commit - робить коміт проекту
  • git commit (-m) - аргумент коміта з допомогою якого можна зробити опис( що було змінено або додано в цьому коміті)
  • git log - показує історію комітів(те що робили в Git)
  • git checkout - переключається на вибрану гілку і оновлює робочу папку до її стану
  • git branch - виводить всі доступні коміти
  • git push - пушить(відправляє) коміт з локального репозиторію у віддалений
  • git pull - завантажує дані з віддаленого репозиторію у локальний
  • git merge - з'єднує дві гілки (або більше) в поточну
  • git clone - завантажує репозиторій та його файли зі всією історією змін
  • .gitignore - файл, який приховує певні файли від системи Git

Основной синтаксис:

  • Каждая строка - отдельный шаблон
  • Пустые строки игнорируются
  • Строки начинающиеся с # являются комментариями
  • Символ слеша "/" в начале строки указывает на текущую папку (где лежит .gitignore)
  • Звёздочка(*) заменяет любое количество символов
  • Две звёздочки(**) используются для указания всех подпапок.
  • Восклицательный знак(!) в начале строки инвертирует шаблон (используется для исключений)
  • Для экранирования спецсимволов используется обратный слэш "\". Для игнорирования всей директории, правило должно оканчиваться на слэш(/), в противном случае правило считается именем файла.

Advanced:

  • git revert - скасовує коміт й створює новий, який видаляє зміни, збережені небажаним
  • git reset - відмінить всі коміти, залишаючи всі зміни в робочій папці
  • git rebase - об'єднує зміни двох гілок
  • git cherry-pick - бере зміни внесені будь-яким комітом й застосовує у вигляді нового коміта
  • git stash - тимчасово збереже всі не збережені зміни
